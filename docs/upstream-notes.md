# Upstream Notes

The upstream FingerprintJS repository is maintained by Fingerprint. Respect the maintainers' contribution workflow.

Before opening a pull request:

- Search existing issues and discussions.
- Prefer a focused discussion if the change is not tied to a reviewed issue.
- Keep changes narrow.
- Do not change playground code unless maintainers ask for it.
- Add tests for code changes.
- Run the checks described in the upstream `contributing.md`.

Good external contribution candidates:

- a small documentation clarification backed by a real migration case
- a typo or broken link fix
- a minimal reproducible issue with browser/version details

Poor contribution candidates:

- a broad rewrite of migration docs without prior discussion
- new dependencies
- subjective style changes
- application-specific patterns that belong in this lab instead of upstream
